## Layer Principles

1. **Validate before build.** Concierge sprint protocols (L2-01) and validation criteria (L2-04) are written first because they block execution. Design docs (L2-02, L2-03) are informed by sprint learnings.
2. **Operational safety first.** All concierge protocols include data consent, disclosure obligations, and user privacy protections. No session runs without these safeguards.
3. **Per-sprint review cadence.** Every sprint cycle triggers a review of sprint-related docs. Learnings from one sprint inform the next.
4. **Pathfinder owns sprint execution.** Concierge playbook and validation framework are DRId by Pathfinder. Design identity is DRId by Craftsman.
