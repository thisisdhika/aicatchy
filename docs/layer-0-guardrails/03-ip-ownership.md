# IP Ownership — Rights & Assignment

*Clarify who owns what — founder IP, contractor IP, AI-generated recommendations, and third-party content.*

---

## 1. Founder IP

| Asset | Owner | Notes |
|-------|-------|-------|
| Pre-existing IP (ideas, research, brand concepts, business plans) | Founder(s) individually | Brought into the venture under an exclusive, perpetual, royalty-free license to AICatchy |
| IP developed within AICatchy (outfit formula library, outfit formulas, code, brand assets, business processes) | AICatchy (the entity) | Assigned to the entity upon formation. If no entity exists yet, all founders sign a joint IP assignment agreement. |
| Company name, domain, trademarks | AICatchy (the entity) | Register under entity name. Founder(s) may not use the name for competing purposes. |
| Investor deck, strategy docs, financial models | AICatchy (the entity) | Created for the venture; owned by the entity. |

**Founder departure:** If a founder leaves, they retain no rights to AICatchy-developed IP. Pre-existing IP licensed to AICatchy remains under the original license terms. Departing founder cannot use AICatchy formulas, data, or brand for competitive purposes.

## 2. Contractor / Stylist IP

### 2.1 Work-for-Hire Assignment

All contractors (stylists, developers, designers, content writers) must sign a **Work-for-Hire Agreement** before starting work.

| Clause | Standard term |
|--------|---------------|
| Definition of "Work Product" | All deliverables created within the engagement: outfit formulas, styling guides, code, designs, editorial copy, spreadsheets, training data |
| Assignment | All right, title, and interest in Work Product automatically vests in AICatchy upon creation |
| Waiver of moral rights | Contractor waives moral rights to the extent permitted by Indonesian law (UU Hak Cipta 2014) |
| Pre-existing IP license | If contractor uses their own IP (e.g., a proprietary styling template), they grant AICatchy an irrevocable, perpetual, royalty-free license |
| No further compensation | Work-for-Hire fee is full and final consideration for the assignment |
| Third-party materials | Contractor warrants that Work Product does not infringe third-party IP. Contractor indemnifies AICatchy for any breach. |

### 2.2 Payment → IP Transfer Sequence

```
Contractor engagement letter signed → Work begins
    → Deliverable submitted
    → AICatchy reviews and accepts
    → Payment released (within 7 days of acceptance)
    → IP assignment effective from first submission, not from payment
```

**IP assignment is effective upon creation, not upon payment.** Payment is the consideration that makes the assignment enforceable. If the contractor is not paid, they have a claim for payment — not a right to revoke IP.

### 2.3 Contractor Types

| Type | IP concern | Mechanism |
|------|------------|-----------|
| Stylist (outfit formulas) | Core IP — the outfit formula library | WFH agreement with explicit formula ownership assignment |
| Developer (code) | Application code, API integrations | WFH agreement; standard in developer contract |
| Designer (UI/graphic) | Brand assets, design system, editorial card templates | WFH agreement; confirm no pre-existing templates used |
| Writer (editorial) | Editorial copy, styling rationale templates | WFH agreement |
| Tester (concierge sprint participants) | Feedback, pain narratives, purchase data | License grant (not assignment) — see §4 |

## 3. AI-Generated Output

### 3.1 Ownership

Under Indonesian copyright law (UU No. 28/2014 tentang Hak Cipta), the **creator** of a work is the person who produces it. For AI-generated output where the AI is a tool:

| Output type | Owner | Rationale |
|-------------|-------|-----------|
| Outfit recommendations generated by AI | AICatchy (the entity) | AI is AICatchy's tool; output is produced as part of AICatchy's service. Human-in-the-loop (styled formulas + AI curation) supports AICatchy's authorship claim. |
| Styling rationale text (AI-written) | AICatchy (the entity) | Written by AI within AICatchy's editorial framework. |
| Product collages / mood images (AI-composed) | AICatchy (the entity) | Composed using AICatchy's designated assets and product data. |

**Caveat:** Indonesian and international law around AI-generated works is evolving. This position will be reviewed when:
- A court issues a ruling on AI-generated work ownership in Indonesia
- UU PDP / copyright amendments address AI output explicitly
- AICatchy receives any IP challenge

### 3.2 Third-Party AI Models

| Model | IP term | Implication |
|-------|---------|-------------|
| OpenAI API | OpenAI does not claim ownership of model output (OpenAI ToS §3 — "Your Content") | AICatchy owns the output. OpenAI receives a license to operate the service. |
| Anthropic API | Anthropic does not claim ownership of model output (Anthropic ToS — "Your Content") | Same as above. |

**No training on AICatchy data:** Both OpenAI and Anthropic commit not to use API-submitted data for model training or improvement (OpenAI API data usage opt-out; Anthropic default for API). AICatchy has opted out of training data usage.

## 4. User-Generated Content

### 4.1 What Users Generate

- Occasion descriptions (free text)
- Vibe keywords
- Saved looks (browser localStorage)
- Shared verdicts (WhatsApp/Instagram — screenshots and text)
- Feedback, ratings, survey responses
- (Future) Styling preference feedback — "this works," "not my style"

### 4.2 License Grant

Users grant AICatchy a **non-exclusive, royalty-free, worldwide, perpetual license** to use shared verdicts and feedback for:

- Product improvement (training outfit formula library)
- Marketing (testimonials, case studies — with attribution only if explicitly consented)
- Analytics and trend analysis
- Internal quality assessment

**What the license does NOT cover:**
- Selling user content to third parties
- Using user identity or likeness without separate consent
- Creating a public profile from user data (AICatchy has no profiles)

### 4.3 User Rights

- Users can request removal of their shared verdict from AICatchy's records by emailing [privacy@aicatchy.com]
- AICatchy will respond within 7 days
- Removal does not affect rights granted under the license for data already used in aggregated/anonymized form

## 5. Third-Party Content

| Content type | Source | Legal basis | Limits |
|-------------|--------|-------------|--------|
| Product images | Shopee or merchant product listings | Fair use / API terms (search result thumbnails) | Must link back to source; cannot assert ownership |
| Product descriptions | Shopee or merchant product feeds | API terms — displayed as-is | Cannot modify significantly; may truncate and cite |
| Brand logos | Product pages | Fair use / nominative use | Only used in recommendation context; no standalone use |
| Price data | Shopee or merchant feeds | API terms — cache limited | Must refresh per caching rules (L3-03 §6) |

**Attribution:** All third-party content is displayed with source attribution (product link to original listing). No third-party content is used outside the recommendation context.

**No scraping:** AICatchy uses official affiliate APIs. Web scraping for product data is not permitted and would violate platform terms.

## 6. Open Source

| Component | License | Usage | Compliance action |
|-----------|---------|-------|-------------------|
| React (or framework) | MIT | Core frontend framework | Attribution in credits |
| Tailwind / CSS framework | MIT | Styling | Attribution if required |
| Backend libraries | MIT / Apache 2.0 | Standard dependencies | Review per library |
| OpenAI SDK | MIT | AI integration | Attribution in credits |
| Any forked/modified component | Per original license | If modified, same license applies | Maintain notice file |

**Open source compliance:**
1. Maintain a `licenses/` directory with copies of all dependency licenses
2. Include attribution in application footer or credits page
3. If AICatchy modifies and distributes a GPL-licensed component, the modified version must also be GPL
4. Review dependencies on every major version bump for license changes

## 7. Assignment Mechanism

### 7.1 Contractor IP Transfer Process

```
Before work:
  1. Contractor receives Work-for-Hire Agreement (template in ../templates/wfh-agreement.md)
  2. Agreement signed by both parties
  3. Contractor added to contractor register (every engagement tracked)

During work:
  4. Deliverables stored in AICatchy's designated repository
  5. AICatchy reviews and accepts deliverable

After work:
  6. Payment processed (within 7 days of acceptance)
  7. IP assignment confirmation recorded in contractor register
  8. Deliverable integrated into AICatchy IP pool
```

### 7.2 Founder IP Transfer Process

```
On entity formation:
  1. All founders execute IP Assignment Agreement
  2. Schedule A: list all pre-existing IP contributed to the venture
  3. Schedule B: list all IP developed within the venture
  4. Agreement filed with company records
```

### 7.3 Record-Keeping

All IP assignments are recorded in `.omp/context/ip-registry.md` with:

| Field | Required |
|-------|----------|
| Date of assignment | Yes |
| Assignor name | Yes |
| IP description (what was assigned) | Yes |
| Agreement reference | Yes (link to signed agreement) |
| Consideration (payment, equity, or other) | Yes |
| Witness / approval | Yes (Steward or Pathfinder) |
